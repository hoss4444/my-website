## General Project Principles
- **Clean Code:** Prioritize readable, semantic HTML and modular CSS/JS.
- **Performance:** Always look for ways to optimize assets and reduce file sizes.
- **Mobile First:** All UI changes must be tested for responsiveness across all breakpoints.

## Development Standards
- **Zero-Hallucination:** If a library or dependency is not currently in the repo, do not use it without asking first.
- **State Management:** Keep logic decoupled from the UI layer where possible.
- **Documentation:** Every new function or complex logic block must include concise comments.

## Interaction Protocol
- Before writing code, provide a step-by-step implementation plan.
- If a task involves visual changes, describe the intended UI/UX impact.
- Check for existing `.md` files in the `/skills` folder before starting specialized tasks.

## Domain Expertise & Skills
This repository uses a modular skills system. Before performing specialized work, refer to the relevant guide in the `/skills` directory:

- For UI/UX and Layout: @skills/UI-UX.md
